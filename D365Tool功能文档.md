# D365 开发工具使用指南 — 一个专为 D365 开发者打造的全能工具箱

## 前言

做 Dynamics 365 开发的兄弟应该都有同感：日常工作中很多操作都在反复做 —— 创建字段、注册插件、部署解决方案、查权限、翻译多语言标签……每次都要在浏览器里点来点去，效率确实不高。

这个工具就是把这些高频操作整合到一个桌面程序里，**不用每次打开浏览器登录 D365**，直接在工具里一键完成。支持 Dev/UAT 多环境切换，所有操作都有日志记录。

## 一、准备工作 — 配置连接

拿到工具后第一件事就是配置你的 D365 环境连接信息。

### 1.1 编辑配置文件

打开 `config.json`，填上你的认证信息：

```json
{
  "tenant_id": "你的租户ID",
  "client_id": "你的应用注册ID",
  "client_secret": "你的客户端密钥",
  "org_url": "https://你的组织.crm5.dynamics.com",
  "publisher_prefix": "mcs"
}
```

如果你有多个环境，可以配置 `environments` 数组：

```json
"environments": [
  {
    "name": "UAT",
    "org_url": "https://uat-org.crm5.dynamics.com",
    "client_id": "...",
    "client_secret": "..."
  }
]
```

> **小提示**：如果 Dev 和 UAT 用的同一个 App Registration，environments 里可以不填 client_id/client_secret，工具会自动继承顶层的配置。

### 1.2 在工具界面配置

启动工具后默认进入「连接配置」面板：

![连接配置界面](这里放你的截图)

填写或修改配置后点击「保存配置」，工具会写入 config.json 文件。

## 二、字段创建 — 批量创建实体字段

这是最常用的功能之一。**不需要每次在浏览器里新建字段了。**

### 使用步骤

**第一步：选择解决方案和实体**

在「解决方案」输入框输入关键字搜索，选好后会自动填充解决方案唯一名。然后输入实体的逻辑名。

**第二步：配置字段信息**

![字段创建界面](这里放你的截图)

填写字段信息：
- **字段逻辑名** — 小写字母开头，只能包含小写字母、数字和下划线（如 `mcs_myfield`）
- **字段架构名** — 大小写字母开头（如 `mcs_MyField`）
- **显示名称** — 用户看到的字段名称
- **字段类型** — 支持以下类型：

| 类型 | 说明 | 额外配置 |
|------|------|---------|
| 单行文本 | 普通文本 | 可设置最大长度 |
| 多行文本 | 长文本 | 可设置最大长度 |
| 整数 | 整数值 | |
| 小数 | 十进制数 | |
| 浮点数 | 浮点数 | |
| 货币 | 金额类型 | |
| 日期时间 | 日期+时间 | |
| 是/否 | 布尔值 | |
| 下拉选项 | 选项集 | 支持本地或全局选项集 |
| 查找 | 关联其他实体 | 需选目标实体 |
| 文件 | 附件类型 | |

**第三步：点击「创建字段」**

工具会自动调用 D365 Web API 创建字段。创建成功后会在下方日志显示结果。

> **注意事项**：
> - 字段逻辑名必须以配置的发布者前缀（如 `mcs_`）开头
> - 如果使用了全局选项集，需要先「加载全局下拉」获取选项集列表
> - 本地下拉选项的格式是每行 `值:标签`，如 `1:待处理`

## 三、实体翻译 — 批量翻译字段显示标签

在 D365 国际化的项目中，字段的显示名称需要翻译成多种语言。这个功能就是干这个的。

### 支持的翻译引擎

工具内置了两个翻译引擎：

- **百度翻译 API** — 覆盖大部分语言
- **有道智云 API** — 专门处理印尼语、马来语、意大利语

> 为什么要用两个 API？因为实测百度对印尼语和马来语的翻译质量不如有道，所以工具会自动路由：**印尼语、马来语、意大利语走有道，其他语言走百度。**

### 配置 API 密钥

在 `config.json` 中配置：

```json
{
  "baidu_app_id": "你的百度APP ID",
  "baidu_secret_key": "你的百度密钥",
  "youdao_app_key": "你的有道APP Key",
  "youdao_app_secret": "你的有道密钥"
}
```

### 使用步骤

![实体翻译界面](这里放你的截图)

1. 在左侧**勾选**需要翻译的目标语言（默认全不选，避免误操作）
2. 在上方输入框输入要翻译的词语，**每行一个**
3. 点击「一键翻译」
4. 右侧表格显示翻译结果，列标题带 LCID 编码（如 `英语 [1033]`）

**复制结果**：选中行后点击「一键复制」，直接粘贴到 Excel，每列对应一种语言。

**批量导入翻译**：还支持导入 D365 标准的 `CrmTranslations.xml` 文件，自动解析并翻译后导出，非常适合大批量翻译场景。

### 支持的语言（13种）

| 语言 | D365 LCID | 使用引擎 |
|------|-----------|---------|
| 英语 | 1033 | 百度 |
| 德语 | 1031 | 百度 |
| 法语 | 1036 | 百度 |
| 西班牙语 | 3082 | 百度 |
| 俄语 | 1049 | 百度 |
| 意大利语 | 1040 | **有道** |
| 葡萄牙语 | 1046 | 百度 |
| 泰语 | 1054 | 百度 |
| 越南语 | 1066 | 百度 |
| 阿拉伯语 | 1025 | 百度 |
| 印度尼西亚语 | 1057 | **有道** |
| 马来西亚语 | 1086 | **有道** |
| 汉语 | 2052 | 百度 |

## 四、插件注册 — 管理 D365 插件

可以查看、注册、更新和删除插件程序集。

### 界面概览

![插件注册界面](这里放你的截图)

界面分为三部分：
- **左侧树**：程序集 → 插件类型 → 步骤的层级结构
- **右下详情**：选中节点的详细信息（属性面板）
- **顶部操作栏**：各种操作按钮

### 支持的操作

| 操作 | 说明 |
|------|------|
| **注册程序集** | 选择 .dll 文件，填写名称和隔离模式，一键注册 |
| **更新程序集** | 选择新的 .dll 文件替换现有程序集 |
| **注册插件类型** | 在选定程序集下注册新类型 |
| **注册步骤** | 配置消息（Create/Update/Delete...）、实体、阶段、模式、Rank 等 |
| **启用/禁用** | 切换步骤的启用状态 |
| **注销** | 删除程序集/类型/步骤 |

> 注意：工具默认只展示指定程序集（如 `SanyD365.D365Extension.Sales`），不会加载所有插件，避免树结构过于庞大。

## 五、实体权限管理 — 可视化编辑权限矩阵

这是权限管理的核心功能，**以矩阵形式直观展示每个安全角色对实体的 8 种权限**。

### 权限矩阵说明

![权限矩阵界面](这里放你的截图)

矩阵的每一列对应一种操作权限：

| 权限 | 说明 | 可选深度 |
|------|------|---------|
| 创建 | 能否创建记录 | 无→用户→部门→子部门→组织 |
| 读 | 能否读取记录 | 同上 |
| 写 | 能否修改记录 | 同上 |
| 删除 | 能否删除记录 | 同上 |
| 追加 | 能否添加关联记录 | 同上 |
| 追加到 | 能否被关联 | 同上 |
| 分派 | 能否转移记录所有权 | 同上 |
| 共享 | 能否共享记录 | 同上 |

**操作方式**：点击角色行中的某个权限图标，循环切换权限深度。修改后点击「保存」，工具会自动对比变更并调用 D365 API 更新。

**复制权限**：如果你有两个实体需要设置相同的权限矩阵，可以先加载一个实体的权限，输入目标实体名，然后「从来源表复制」。

## 六、脚本 JS 调试 — 本地文件替换调试

D365 的 JS/HTML WebResource 调试一直是个痛点 —— 改一行代码要上传→发布→刷新，非常浪费时间。这个功能通过 **Chrome DevTools Protocol** 实现本地文件实时替换。

### 工作流程

1. **打开 Chrome 远程调试端口**（启动浏览器时加 `--remote-debugging-port=9222`）
2. **在工具中添加替换规则**：匹配需要替换的 URL 和本地文件路径
3. **点击「开始调试」**，工具连接到浏览器
4. **刷新 D365 页面**，匹配的 JS/HTML 会自动使用本地文件内容替换
5. **本地修改代码后保存**，工具自动重新注入（如果勾选了「保存后重新开始调试」）

> **适用场景**：调试 JavaScript WebResource、替换 HTML 资源文件、快速验证代码修改

## 十、常见问题

**Q：启动时报激活窗口？**
A：这是软件的授权机制。需要运行 `generate_activation_code.py` 根据机器码生成激活码。

**Q：翻译时提示 API 配置错误？**
A：检查 `config.json` 中 baidu_app_id / youdao_app_key 等字段是否正确填写。百度翻译和有道翻译都需要在各自的开放平台申请 API 权限。

**Q：插件树加载不出来？**
A：确认工具只加载白名单内的程序集（`SanyD365.D365Extension.Sales` 等），如果程序集名不匹配，需要修改代码中的 `PLUGIN_ASSEMBLY_ALLOWLIST` 常量。

**Q：JS 调试连不上浏览器？**
A：确保 Chrome 以 `--remote-debugging-port=9222` 参数启动，并且没有被防火墙阻止。
